# peer-review

Use this skill when the user asks for a formal peer review, grant review, manuscript critique, revision memo, or journal-style reviewer report.

## Workflow

1. Identify manuscript type, field, target venue, review criteria, and any supplied review form.
2. Summarize the work in 2 to 4 sentences without adding unsupported claims.
3. Review novelty, importance, methods, statistics, reporting standards, ethics, reproducibility, figures/tables, and interpretation.
4. Separate major issues from minor issues. Major issues must affect validity, reproducibility, interpretation, scope, ethics, or publication readiness.
5. Write feedback that is specific, evidence-grounded, and actionable.

## Reference Use

Use:

- `references/common_issues.md` for recurring manuscript problems
- `references/reporting_standards.md` for CONSORT, STROBE, PRISMA, ARRIVE, and related standards

## Output Shape

Return:

- short manuscript summary
- overall recommendation or readiness judgment when requested
- major comments
- minor comments
- reproducibility and data/code availability notes
- reporting guideline gaps
- constructive revision priorities

## Mobile Boundary

This is a documentation-only review skill. Do not run reviewer scripts, local schematic generation, or desktop tooling.
