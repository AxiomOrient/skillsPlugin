# scientific-brainstorming

Use this skill when the user asks for research ideation, early-stage project exploration, research gap discovery, unconventional angles, or experimental design brainstorming.

## Workflow

1. Clarify the research context, domain, constraints, current model, and what kind of novelty the user wants.
2. Generate diverse ideas before evaluating them. Use cross-domain analogies, assumption reversal, scale shifts, constraint changes, and interdisciplinary combinations.
3. Cluster the ideas by mechanism, method, dataset, population, or practical path.
4. Evaluate the strongest candidates for novelty, feasibility, testability, cost, risk, and required collaborators.
5. End with concrete next actions: literature checks, pilot experiments, data needed, and decision points.

## Reference Use

Read `references/brainstorming_methods.md` when the user requests a named ideation method or when the session needs more structure.

## Output Shape

Return:

- clarified research frame
- idea set grouped by theme
- strongest 3 to 5 directions
- first test or literature check for each direction
- assumptions and risks to revisit

## Mobile Boundary

This is a documentation-only AMA skill. Do not ask AMA to run Python, shell, notebooks, desktop tools, package managers, or local scripts.
