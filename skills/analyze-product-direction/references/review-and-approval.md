# Review Pass

Before presenting the direction, run one adversarial review pass.

## Review Dimensions

- Completeness
- Consistency
- Clarity
- Scope
- Feasibility

## Workflow

1. Run a strict review pass against the proposed direction.
2. Fix the issues you accept.
3. Re-run the review.
4. Stop after 3 iterations max.

If delegation is explicitly available and allowed, a fresh reviewer may perform the review. Otherwise, do a separate self-review pass.

If the same issues repeat across iterations, stop looping and present the direction with the unresolved concerns called out explicitly.

## Handoff

After the review pass, close with:

- the recommended direction,
- the single most important next real-world validation step,
- the best next downstream activity.
